# FOF 管理平台 - 目录结构说明

## 📁 项目目录结构

```
fof/
├── .git/                    # Git版本控制
├── .github/                 # GitHub配置
├── assets/                  # 资源文件（图片、PDF、Excel等）
├── backend/                 # 后端代码
├── data/                    # 数据文件（CSV、JSON等）
├── database/                # 数据库文件
├── deploy/                  # 部署脚本和配置
├── docker/                  # Docker配置
├── docs/                    # 文档
├── frontend/                # 前端代码
├── logs/                    # 日志文件

├── tests/                   # 测试文件

```

## 📋 整理说明

### 整理前
- 根目录文件混乱（50+个文件）
- 文档、测试、工具混在一起
- 难以查找和维护

### 整理后
- 根目录只保留核心文件（7个）
- 文件按类型分类到子目录
- 结构清晰，易于维护

### 整理原则
1. ✅ 按功能分类（文档、测试、工具、资源）
2. ✅ 保持目录结构扁平
3. ✅ 文件命名清晰
4. ✅ 便于查找和维护
