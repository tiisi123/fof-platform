# FOF平台部署问题修复指南

## 问题：量化库安装失败（scs编译错误）

### 错误信息
```
ERROR: Problem encountered: OpenBLAS or Netlib BLAS/CBLAS is required on all platforms, and was not found.
```

### 原因分析
- `scs` 包需要编译，依赖 OpenBLAS/BLAS 系统库
- 服务器缺少必要的编译工具和数学库

---

## 解决方案

### 方案1：安装系统依赖（推荐 - 完整功能）

适用于需要使用量化分析功能的生产环境。

#### CentOS/RHEL 系统
```bash
# 安装开发工具
yum groupinstall -y "Development Tools"

# 安装数学库
yum install -y openblas-devel lapack-devel cmake

# 然后重新运行部署脚本
cd /root/web/fof/deploy
bash 一键部署.sh
```

#### Ubuntu/Debian 系统
```bash
# 更新包列表
apt-get update

# 安装依赖
apt-get install -y build-essential libopenblas-dev liblapack-dev cmake

# 然后重新运行部署脚本
cd /root/web/fof/deploy
bash 一键部署.sh
```

---

### 方案2：禁用量化库（快速部署）

适用于快速部署或不需要量化分析功能的场景。

#### 修改部署脚本
编辑 `一键部署.sh`，在文件开头的配置部分修改：

```bash
# 配置
PROJECT_DIR="/root/web/fof"
BACKUP_DIR="/root/backups"
BACKEND_PORT="8506"
FRONTEND_PORT="8507"
GIT_REPO="https://gitee.com/ts_tiisi/fof_v2.git"
CONDA_ENV="py310fof"
CONDA_PATH="/root/anaconda3"
INSTALL_QUANT_LIBS="no"  # 改为 no 禁用量化库
```

然后重新运行部署脚本。

---

### 方案3：手动安装（精细控制）

如果自动脚本失败，可以手动执行：

```bash
# 1. 激活conda环境
source /root/anaconda3/etc/profile.d/conda.sh
conda activate py310fof

# 2. 进入项目目录
cd /root/web/fof

# 3. 安装基础依赖
pip install -r backend/requirements.txt

# 4. 尝试安装量化库（可选）
# 如果失败，可以跳过这一步
pip install -r backend/requirements-open-source.txt || echo "量化库安装失败，将使用基础功能"

# 5. 继续其他部署步骤...
```

## 功能影响说明

### 禁用量化库后的影响

如果选择方案2（禁用量化库），以下功能将不可用：

- ❌ PyPortfolioOpt - 投资组合优化
- ❌ Riskfolio-Lib - 风险预算/CVaR优化
- ❌ quantstats - 性能报告生成
- ❌ akshare - 市场数据获取
- ❌ OpenBB - 多源市场数据

### 仍然可用的功能

- ✅ 基金管理
- ✅ NAV数据处理
- ✅ 文档管理
- ✅ 用户权限管理
- ✅ 待办任务
- ✅ 审计日志
- ✅ Dashboard展示
- ✅ 基础数据分析（使用内置引擎）

## 常见问题

### Q1: 为什么会出现这个错误？
A: 量化库依赖的 `scs` 包需要从源码编译，需要系统安装 OpenBLAS 等数学库。

### Q2: 不安装量化库会影响核心功能吗？
A: 不会。核心的基金管理、NAV处理、文档管理等功能不受影响。

### Q3: 后续可以再安装量化库吗？
A: 可以。先安装系统依赖，然后运行：
```bash
conda activate py310fof
cd /root/web/fof
pip install -r backend/requirements-open-source.txt
systemctl restart fof-backend
```

### Q4: 如何确认量化库是否安装成功？
A: 运行以下命令测试：
```bash
conda activate py310fof
python -c "import riskfolio; print('量化库安装成功')"
```
