# FOF管理平台业务测试说明

## 测试脚本说明

本目录提供了两个业务功能测试脚本，用于验证FOF管理平台的核心功能模块。

### 1. Shell脚本版本 (业务测试.sh)

**特点:**
- 轻量级，无需额外依赖
- 使用curl进行HTTP请求
- 适合快速验证和CI/CD集成

**使用方法:**
```bash
cd /root/web/fof/deploy
./业务测试.sh
```

### 2. Python脚本版本 (test_business.py)

**特点:**
- 更详细的测试报告
- 更好的错误处理
- 支持JSON数据解析
- 显示测试耗时统计

**使用方法:**
```bash
cd /root/web/fof/deploy
python3 test_business.py
```

或者直接执行:
```bash
./test_business.py
```

## 测试前提条件

### 1. 服务运行状态

确保后端服务正在运行:
```bash
systemctl status fof-backend
```

如果服务未运行，启动服务:
```bash
systemctl start fof-backend
```

### 2. 网络连接

测试脚本默认连接到 `http://localhost:8506`

如果需要测试远程服务器，修改脚本中的配置:
```bash
# Shell脚本
BACKEND_URL="http://your-server-ip:8506"

# Python脚本
BACKEND_URL = "http://your-server-ip:8506"
```

### 3. 默认账号

测试使用默认管理员账号:
- 用户名: `admin`
- 密码: `admin123`

如果修改了默认密码，需要更新脚本中的配置。

## 常见问题排查

### 1. 服务无响应

**问题:** `[✗] 后端服务无响应`

**解决方案:**
```bash
# 检查服务状态
systemctl status fof-backend

# 查看服务日志
journalctl -u fof-backend -n 50

# 重启服务
systemctl restart fof-backend
```

### 2. 认证失败

**问题:** `[✗] 管理员登录失败`

**解决方案:**
- 检查默认密码是否已修改
- 查看 `backend/.env` 中的 `ADMIN_DEFAULT_PASSWORD` 配置
- 确认数据库中管理员账号存在

### 3. 数据库错误

**问题:** 多个接口返回 500 错误

**解决方案:**
```bash
# 检查数据库文件
ls -lh /root/web/fof/backend/fof.db

# 重新初始化数据库
cd /root/web/fof/backend
python scripts/regenerate_tasks.py
python scripts/add_dashboard_indexes.py
```

### 4. 端口占用

**问题:** 服务启动失败，端口被占用

**解决方案:**
```bash
# 查看端口占用
netstat -tlnp | grep 8506

# 杀死占用进程
kill -9 <PID>

# 重启服务
systemctl restart fof-backend
```

### 添加新的测试用例

在Python脚本中添加新的测试方法:

```python
def test_new_module(self):
    """新模块测试"""
    print("\n" + "="*50)
    self.log_info("15. 新模块测试")
    print("="*50)
    
    self.test_start()
    self.log_info("测试新功能...")
    
    try:
        response = requests.get(
            f"{API_BASE}/new-endpoint",
            headers={"Authorization": f"Bearer {self.token}"},
            timeout=10
        )
        
        if response.status_code == 200:
            self.log_success("新功能测试成功")
        else:
            self.log_error(f"新功能测试失败 (状态码: {response.status_code})")
    except Exception as e:
        self.log_error("新功能请求异常", str(e))
```

然后在 `run_all_tests()` 方法中调用:

```python

```

## 测试报告

### 生成HTML报告

可以扩展Python脚本生成HTML格式的测试报告:

```python
def generate_html_report(self):
    """生成HTML测试报告"""
    html = f"""
    <!DOCTYPE html>
    <html>
    <head>
        <title>FOF平台测试报告</title>
        <style>
            body {{ font-family: Arial, sans-serif; margin: 20px; }}
            .pass {{ color: green; }}
            .fail {{ color: red; }}
        </style>
    </head>
    <body>
        <h1>FOF管理平台测试报告</h1>
        <p>测试时间: {datetime.now()}</p>
        <p>总测试数: {self.total_tests}</p>
        <p class="pass">通过: {self.passed_tests}</p>
        <p class="fail">失败: {self.failed_tests}</p>
        <h2>测试详情</h2>
        <ul>
    """
    
    for result in self.results:
        status_class = "pass" if result[0] == "PASS" else "fail"
        html += f'<li class="{status_class}">{result[1]}</li>'
    
    html += """
        </ul>
    </body>
    </html>
    """
    
    with open("test_report.html", "w", encoding="utf-8") as f:
        f.write(html)
    
    self.log_info("HTML报告已生成: test_report.html")
```
