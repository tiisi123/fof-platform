## 🛠️ 常见问题

### Q1: 应该使用哪个部署脚本？

**A:** 推荐使用 `简化部署-v2.sh`，它解决了所有已知问题。

---

### Q2: 部署失败怎么办？

**A:** 
1. 查看错误日志：`journalctl -u fof-backend -n 50`
2. 使用诊断工具：`bash 诊断工具.sh`
3. 根据错误使用对应修复脚本

---

### Q3: 如何更新部署？

**A:**
```bash
cd /root/web/fof
git pull origin master
bash deploy/简化部署-v2.sh
```

---

### Q4: 如何卸载？

**A:**
```bash
# 停止服务
systemctl stop fof-backend fof-frontend
systemctl disable fof-backend fof-frontend

# 删除服务文件
rm /etc/systemd/system/fof-*.service
systemctl daemon-reload

# 删除项目（可选）
rm -rf /root/web/fof
```
