# FOF管理平台业务测试原理说明

## 📖 测试原理概述

这个业务测试是一个**自动化API接口测试**，通过模拟真实用户的操作流程，验证服务器上各个业务模块的功能是否正常。

## 🔍 测试工作流程

### 1. 测试架构

```
测试脚本 (本地/客户端)
    ↓ HTTP请求
服务器 (47.116.187.192:8506)
    ↓ 处理请求
后端API (FastAPI)
    ↓ 数据库查询
SQLite数据库 (fof.db)
    ↓ 返回数据
响应返回给测试脚本
```

### 2. 测试步骤详解

#### 步骤1: 连接测试
```python
def test_connection(self) -> bool:
```

**做什么：**
- 向服务器发送 `GET /health` 请求

#### 步骤2: 登录认证
```python
def test_login(self) -> bool:
    response = requests.post(
        f"{API_BASE}/auth/login",
        data={"username": "admin", "password": "admin123"},
        timeout=10
    )
    self.token = response.json().get("access_token")
```

**做什么：**
- 使用管理员账号登录
- 获取JWT访问令牌(Token)

**验证什么：**
- ✅ 用户认证系统正常
- ✅ 数据库用户表可访问
- ✅ JWT令牌生成正常

**实际请求：**
```bash
curl -X POST http://47.116.187.192:8506/api/v1/auth/login \
  -d "username=admin&password=admin123"
```

**预期响应：**
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer",
  "expires_in": 604800
}
```

**Token的作用：**
- 后续所有请求都需要携带这个Token
- 服务器通过Token识别用户身份
- Token包含用户权限信息

## 🔧 测试技术细节

### HTTP请求方法
```python
# GET请求 - 查询数据
requests.get(url, params={...}, headers={...})

# POST请求 - 提交数据
requests.post(url, data={...}, headers={...})
```

### 认证机制
```python
# 1. 登录获取Token
token = login_response.json()["access_token"]

# 2. 后续请求携带Token
headers = {"Authorization": f"Bearer {token}"}
```

### 错误处理
```python
try:
    response = requests.get(url, timeout=10)
    if response.status_code == 200:
        # 成功
    else:
        # 失败 - 记录状态码
except requests.exceptions.ConnectionError:
    # 网络连接错误
except Exception as e:
    # 其他异常
```

## 🆚 与手工测试的对比

### 手工测试
1. 打开浏览器
2. 访问 http://47.116.187.192:8507
3. 输入用户名密码登录
4. 点击产品管理菜单
5. 查看产品列表是否显示
6. 重复以上步骤测试其他模块

**缺点：**
- ❌ 耗时长（每次10-20分钟）
- ❌ 容易遗漏
- ❌ 无法频繁执行
- ❌ 难以自动化

### 自动化测试
1. 运行 `python test_remote_server.py`
2. 等待10秒
3. 查看测试报告

**优点：**
- ✅ 快速（10秒完成）
- ✅ 全面（覆盖所有模块）
- ✅ 可重复执行
- ✅ 可集成到CI/CD
