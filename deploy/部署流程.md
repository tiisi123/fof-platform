# FOF平台完整部署流程

## 推荐部署流程（分步执行）

### 第一步：预安装系统依赖

```bash
cd /root/web/fof/deploy

# 给脚本执行权限
chmod +x 0-预安装依赖.sh

# 运行预安装脚本
bash 0-预安装依赖.sh
```

这个脚本会安装：
- gcc/g++ 编译器
- OpenBLAS 数学库
- LAPACK 线性代数库
- CMake 构建工具

**预计时间：** 3-5分钟

---

### 第二步：运行一键部署

```bash
cd /root/web/fof/deploy

# 给脚本执行权限
chmod +x 一键部署.sh

# 运行部署脚本
bash 一键部署.sh
```

部署脚本会自动：
1. ✅ 检查 conda 环境
2. ✅ 备份现有项目
3. ✅ 停止旧服务
4. ✅ 更新代码
5. ✅ 安装 Python 依赖（包括量化库）
6. ✅ 构建前端
7. ✅ 初始化数据库
8. ✅ 配置并启动服务

**预计时间：** 10-15分钟

---

## 快速部署（不安装量化库）

如果不需要量化分析功能，可以跳过第一步：

```bash
cd /root/web/fof/deploy

# 编辑部署脚本，设置 INSTALL_QUANT_LIBS="no"
vim 一键部署.sh
# 或
nano 一键部署.sh

# 找到这一行并修改：
# INSTALL_QUANT_LIBS="no"

# 然后运行
bash 一键部署.sh
```

**预计时间：** 5-8分钟

## 手动部署（完全控制）

如果自动脚本有问题，可以手动执行：

### 1. 安装系统依赖（CentOS/RHEL）

```bash
# 安装开发工具
yum groupinstall -y "Development Tools"

# 安装数学库
yum install -y openblas-devel lapack-devel cmake
```

### 2. 激活 conda 环境

```bash
source /root/anaconda3/etc/profile.d/conda.sh
conda activate py310fof
```

### 3. 进入项目目录

```bash
cd /root/web/fof
```

### 4. 安装 Python 依赖

```bash
# 安装基础依赖
pip install -r backend/requirements.txt

# 安装量化库（可选）
pip install -r backend/requirements-open-source.txt
```

### 5. 构建前端

```bash
cd frontend
npm install
npm run build
cd ..
```

### 6. 配置环境变量

```bash
# 如果没有 .env 文件
cp backend/.env.example backend/.env

# 编辑配置
vim backend/.env
```

### 7. 初始化数据库

```bash
cd backend

# 生成待办任务
python scripts/regenerate_tasks.py

# 添加索引
python scripts/add_dashboard_indexes.py

cd ..
```

### 8. 配置 systemd 服务

```bash
# 复制服务文件
cp deploy/fof-backend.service /etc/systemd/system/
cp deploy/fof-frontend.service /etc/systemd/system/

# 重新加载
systemctl daemon-reload

# 启用服务
systemctl enable fof-backend fof-frontend

# 启动服务
systemctl start fof-backend fof-frontend
```

### 9. 检查服务状态

```bash
# 查看服务状态
systemctl status fof-backend
systemctl status fof-frontend

# 查看日志
journalctl -u fof-backend -f
```

## 常见问题排查

### 问题1：conda 环境不存在

```bash
# 查看可用环境
conda env list

# 创建环境
conda create -n py310fof python=3.10
```

### 问题2：端口被占用

```bash
# 查看占用端口的进程
lsof -i :8506
lsof -i :8507

# 杀死进程
kill -9 <PID>
```

### 问题3：权限问题

```bash
# 确保目录权限正确
chown -R root:root /root/web/fof

# 确保脚本可执行
chmod +x deploy/*.sh
```

### 问题4：数据库锁定

```bash
# 停止所有服务
systemctl stop fof-backend fof-frontend

# 检查数据库文件
ls -lh backend/fof.db

# 重启服务
systemctl start fof-backend fof-frontend
```
