# 因子归因分析使用指南

## 快速开始

### 1. 查看可用产品

双击运行：
```
查看可归因分析产品.bat
```

或命令行：
```bash
cd backend
python list_attribution_ready_products.py
```

这会显示：
- 可以进行归因分析的产品列表（308个）
- 按净值数据量排序
- 按策略类型统计

### 2. 选择产品进行分析

在前端页面：
1. 打开 "因子归因分析" 页面
2. 从下拉框选择产品（推荐选择净值数据>100条的产品）
3. 可选：设置分析周期
4. 点击 "分析" 按钮

### 3. 查看分析结果

系统会显示：
- **因子暴露雷达图**: 8个因子的暴露情况
- **业绩归因饼图**: 各因子对收益的贡献
- **累计归因走势**: 时间序列上的归因变化
- **因子贡献明细表**: 详细的数值数据
- **风格标签**: 自动生成的风格特征

## 因子说明

### 8个核心因子

1. **市场因子 (Market)**: 整体市场风险敞口
2. **规模因子 (Size)**: 大盘股 vs 小盘股偏好
3. **价值因子 (Value)**: 低估值股票偏好
4. **动量因子 (Momentum)**: 趋势跟踪特征
5. **波动率因子 (Volatility)**: 低波动率股票偏好
6. **质量因子 (Quality)**: 高质量公司偏好
7. **成长因子 (Growth)**: 高成长股票偏好
8. **流动性因子 (Liquidity)**: 流动性风险补偿

### 解读指标

- **因子暴露**: 产品对该因子的敏感度（-1到1）
- **R²**: 模型解释力（越高越好，>0.7为优秀）
- **Alpha**: 超额收益能力（年化）
- **跟踪误差**: 残差波动率

## 补充缺失数据

如果发现某些产品无法分析（净值数据不足），可以尝试补充采集：

双击运行：
```
补充采集净值数据.bat
```

或命令行：
```bash
cd backend
python scripts/补充采集净值数据.py
```

该脚本会：
- 自动识别183个缺失净值的产品
- 重新调用火富牛API获取
- 显示成功/失败统计
- 记录失败原因

**注意**: 
- 某些产品可能确实没有净值数据（已清盘、未公开等）
- 补充采集需要时间（约5-10分钟）
- 建议在非高峰时段运行

## 常见问题

### Q1: 为什么有些产品显示"数据不足"？

**A**: 因子归因分析需要至少30个净值数据点。部分产品在初次采集时未能获取到净值数据，可以运行补充采集脚本尝试重新获取。

### Q2: 分析结果准确吗？

**A**: 当前使用模拟因子数据进行演示。实际应用中需要：
- 接入真实的Barra因子数据（Wind/万得）
- 或使用其他因子数据源
- 配置方式见前端页面的"因子数据来源"说明

### Q3: 如何比较多个产品？

**A**: 使用比较接口：
```
GET /api/v1/attribution/compare?product_ids=1,2,3
```

### Q4: 可以导出分析报告吗？

**A**: 当前版本暂不支持，后续版本会添加PDF/Excel导出功能。
