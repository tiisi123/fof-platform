# 环境配置说明

## 重要提示

⚠️ **敏感文件不要上传到 GitHub！**

以下文件包含敏感信息，已在 `.gitignore` 中排除：
- `backend/.env` - 环境变量配置（包含密钥）
- `backend/fof.db` - 数据库文件（包含业务数据）

## 一、环境变量配置

### 1. 复制模板文件

```bash
cd /root/web/fof/backend
cp .env.example .env
```

---

### 2. 修改配置

编辑 `.env` 文件，修改以下配置：

#### 必须修改的配置

```bash
# 1. 密钥（生成随机字符串）
SECRET_KEY=你的随机密钥
JWT_SECRET_KEY=你的JWT密钥

# 2. CORS 配置（添加你的服务器 IP）
CORS_ORIGINS=["http://localhost:8507","http://你的服务器IP:8507"]

# 3. LLM API 密钥（如果使用 AI 功能）
LLM_API_KEY=你的阿里云百炼API密钥
```

#### 可选配置

```bash
# 数据库（默认使用 SQLite，无需修改）
DATABASE_URL=sqlite:///./fof.db

# 如果使用 MySQL
# DATABASE_URL=mysql+pymysql://用户名:密码@数据库地址:3306/数据库名

# 日志级别
LOG_LEVEL=INFO

# 舆情爬虫（默认禁用）
ENABLE_SENTIMENT_CRAWLER=false
```

---

### 3. 生成随机密钥

```bash
# 方法1：使用 Python
python3 -c "import secrets; print(secrets.token_urlsafe(32))"

# 方法2：使用 openssl
openssl rand -base64 32
```

## 三、数据库备份

### 1. 备份 SQLite 数据库

```bash
# 备份到本地
cp /root/web/fof/backend/fof.db /root/backup/fof.db.$(date +%Y%m%d)

# 备份到远程服务器
scp /root/web/fof/backend/fof.db user@backup-server:/backup/
```

---

### 2. 定期备份脚本

创建 `/root/backup_fof.sh`：

```bash
#!/bin/bash
# FOF 数据库备份脚本

BACKUP_DIR="/root/backup/fof"
DB_FILE="/root/web/fof/backend/fof.db"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
cp $DB_FILE $BACKUP_DIR/fof_$DATE.db

# 压缩备份
gzip $BACKUP_DIR/fof_$DATE.db

# 删除 7 天前的备份
find $BACKUP_DIR -name "fof_*.db.gz" -mtime +7 -delete

echo "备份完成: $BACKUP_DIR/fof_$DATE.db.gz"
```

设置定时任务：

```bash
# 编辑 crontab
crontab -e

# 添加每天凌晨 2 点备份
0 2 * * * /root/backup_fof.sh
```

## 六、安全建议

### 1. 密钥管理

- ✅ 使用强随机密钥（至少 32 字符）
- ✅ 不同环境使用不同密钥
- ✅ 定期更换密钥
- ❌ 不要在代码中硬编码密钥
- ❌ 不要将 `.env` 提交到 Git

---

### 2. 数据库安全

- ✅ 定期备份数据库
- ✅ 备份文件加密存储
- ✅ 限制数据库文件权限
- ❌ 不要将数据库文件上传到 GitHub
- ❌ 不要在公网暴露数据库端口

---

### 3. 访问控制

- ✅ 使用强密码
- ✅ 启用 HTTPS
- ✅ 限制 CORS 来源
- ✅ 定期审查访问日志
- ❌ 不要使用默认密码

## 七、常见问题

### Q1: 忘记修改 SECRET_KEY 怎么办？

**A:** 立即修改并重启服务

```bash
# 1. 生成新密钥
python3 -c "import secrets; print(secrets.token_urlsafe(32))"

# 2. 修改 .env
vim /root/web/fof/backend/.env

# 3. 重启服务
systemctl restart fof-backend
```

---

### Q2: 数据库文件丢失怎么办？

**A:** 从备份恢复

```bash
# 查找最新备份
ls -lh /root/backup/fof/

# 恢复备份
gunzip /root/backup/fof/fof_20260413.db.gz
cp /root/backup/fof/fof_20260413.db /root/web/fof/backend/fof.db

# 重启服务
systemctl restart fof-backend
```

---

### Q3: 如何迁移到新服务器？

**A:** 按照新服务器部署流程

1. 在新服务器克隆代码
2. 复制 `.env` 文件（或重新配置）
3. 复制 `fof.db` 数据库文件
4. 运行部署脚本
5. 验证服务正常

---

### Q4: 如何查看当前配置？

**A:** 查看环境变量文件

```bash
# 查看配置（隐藏敏感信息）
cat /root/web/fof/backend/.env | grep -v "KEY\|PASSWORD"

# 查看完整配置（小心敏感信息）
cat /root/web/fof/backend/.env
```
