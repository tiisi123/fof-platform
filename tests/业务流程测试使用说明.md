## 🎯 两种测试方式

### 方式1：自动化测试（推荐快速验证）

**适用场景**:
- 快速验证系统基本功能
- 回归测试
- 持续集成

**测试范围**:
- ✅ 流程1: 管理人管理流程（10步）
- ✅ 流程2: 产品管理流程（9步）
- ⚠️ 流程3: 净值数据管理流程（需手动）
- ✅ 流程4: 组合管理流程（8步）
- ✅ 流程5: 项目管理流程（9步）
- ⚠️ 流程6-10: 需手动测试

**使用步骤**:

#### Windows系统
```bash
# 1. 双击运行
运行业务流程测试.bat

# 或命令行运行
cd tests
python 业务流程自动化测试.py
```

#### Linux系统
```bash
# 1. 添加执行权限
chmod +x 运行业务流程测试.sh

# 2. 运行测试
./运行业务流程测试.sh

# 或直接运行Python脚本
python3 业务流程自动化测试.py
```

**前置条件**:
1. 安装Python 3.7+
2. 安装Selenium: `pip install selenium`
3. 安装Chrome浏览器
4. 下载ChromeDriver并配置到PATH

**测试结果**:
- 控制台实时显示测试进度
- 自动生成JSON格式测试报告
- 报告文件名: `业务流程测试报告_YYYYMMDD_HHMMSS.json`

## 🔄 推荐测试流程

### 第一次测试（完整验证）
```
1. 先运行自动化测试（30分钟）
   - 快速验证基本功能
   - 发现明显问题
   
2. 再进行手动测试（3-4小时）
   - 深度验证业务流程
   - 测试复杂场景
   - 记录详细问题
```

### 后续测试（回归测试）
```
1. 运行自动化测试（30分钟）
   - 验证修复效果
   - 确保无新问题
   
2. 针对性手动测试（1-2小时）
   - 重点测试修改的功能
   - 验证问题修复
```

## ⚠️ 注意事项

### 自动化测试
1. **浏览器要求**
   - 必须安装Chrome浏览器
   - ChromeDriver版本要与Chrome版本匹配
   - 测试时不要操作浏览器

2. **网络要求**
   - 确保能访问测试环境
   - 网络延迟可能影响测试结果
   - 超时时间设置为10秒

3. **测试数据**
   - 自动生成测试数据（带时间戳）
   - 测试完成后会尝试清理数据
   - 建议定期手动清理测试数据

### 手动测试
1. **测试数据**
   - 使用"测试"前缀标识测试数据
   - 测试完成后及时清理
   - 不要使用生产数据测试

2. **问题记录**
   - 详细记录重现步骤
   - 截图保存关键界面
   - 分类问题严重程度

3. **时间安排**
   - 预留足够时间（3-4小时）
   - 不要中途中断
   - 保持专注

## 📞 常见问题

### Q1: ChromeDriver版本不匹配怎么办？
**A**: 
1. 查看Chrome版本: chrome://version
2. 下载对应版本ChromeDriver: https://chromedriver.chromium.org/
3. 将ChromeDriver放到PATH路径

### Q2: 自动化测试失败怎么办？
**A**:
1. 检查网络连接
2. 确认系统正常运行
3. 查看控制台错误信息
4. 尝试手动测试该流程

### Q3: 如何查看测试报告？
**A**:
- 自动化测试: 打开生成的JSON文件
- 手动测试: 填写Markdown报告模板

### Q4: 测试数据如何清理？
**A**:
- 自动化测试会尝试自动清理
- 手动测试需要手动删除
- 建议定期清理所有测试数据

### Q5: 可以只测试部分流程吗？
**A**:
- 自动化测试: 修改脚本注释掉不需要的流程
- 手动测试: 只测试需要的流程即可
