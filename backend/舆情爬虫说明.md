# 舆情爬虫功能说明

## 当前状态

**默认：已禁用** ❌

原因：
- 数据源可能失效（一直返回 0 条新文章）
- 每 4 小时打印日志，产生噪音
- 实际使用价值不明确

## 功能说明

舆情爬虫会定时从配置的数据源（如东方财富）抓取管理人相关新闻，并自动进行情感分析。

### 数据源

当前配置的数据源：
- ✅ 东方财富搜索 API（已启用）
- ❌ 新浪财经搜索（已禁用）

### 运行机制

1. 启动后延迟 30 秒开始首次爬取
2. 每 4 小时（14400 秒）执行一次
3. 遍历所有活跃管理人
4. 为每个管理人搜索相关新闻
5. 去重后保存到数据库
6. 自动进行情感分析

## 如何启用

### 方法1：修改环境变量（推荐）

编辑 `backend/.env` 文件：

```bash
# 启用舆情爬虫
ENABLE_SENTIMENT_CRAWLER=true

# 可选：修改爬取间隔（秒）
SENTIMENT_CRAWLER_INTERVAL=14400  # 4小时
```

然后重启服务：
```bash
systemctl restart fof-backend
```

### 方法2：手动触发

即使禁用了定时爬取，仍然可以通过 API 手动触发：

```bash
# 为所有管理人爬取
curl -X POST http://localhost:8506/api/v1/sentiment/crawl

# 为单个管理人爬取
curl -X POST http://localhost:8506/api/v1/sentiment/crawl/{manager_id}
```

## 故障排查

### 问题1：一直是 0 条新文章

**可能原因：**
- 数据源 API 变更或失效
- 搜索关键词不匹配
- 网络问题

**解决方案：**
1. 检查数据源 API 是否可用
2. 手动测试单个管理人爬取
3. 查看详细错误日志

### 问题2：爬取失败

**查看日志：**
```bash
journalctl -u fof-backend | grep sentiment_crawler
```

**常见错误：**
- 网络超时：增加 timeout 配置
- API 限流：增加爬取间隔
- 数据格式变化：更新解析逻辑

## 数据源配置

数据源配置在 `app/services/sentiment_crawler.py` 中：

```python
NEWS_SOURCES = [
    {
        "name": "东方财富搜索",
        "type": "eastmoney",
        "base_url": "https://search-api-web.eastmoney.com/search/jsonp",
        "enabled": True,
    },
    {
        "name": "新浪财经搜索",
        "type": "sina",
        "base_url": "https://search.sina.com.cn/news",
        "enabled": False,
    },
]
```

### 添加新数据源

1. 在 `NEWS_SOURCES` 中添加配置
2. 实现对应的 `_fetch_xxx` 方法
3. 在 `_fetch_news` 中调用

## API 接口

### 手动触发爬取

```http
POST /api/v1/sentiment/crawl
```

**响应：**
```json
{
  "total_managers": 17,
  "new_articles": 5,
  "errors": 0
}
```

### 为单个管理人爬取

```http
POST /api/v1/sentiment/crawl/{manager_id}
```

**响应：**
```json
{
  "manager_name": "某某基金",
  "new_articles": 2
}
```

## 性能影响

### 资源消耗

- CPU：低（主要是网络 I/O）
- 内存：低（< 50MB）
- 网络：中等（取决于管理人数量）

### 建议

- 生产环境：根据实际需求决定是否启用
- 开发环境：建议禁用，减少日志噪音
- 如需实时数据：使用手动触发 API
